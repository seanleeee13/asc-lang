1. 변수

2. 값 표현
자연수: []+++ (+ 개수가 수)
-> a+와 같이 썼을 때 + 연산자는 1을 늘리는 연산을 하고, 빈 리스트는 0으로 취급됨
음의 정수: []---
-> - 연산자는 1을 줄임
0: []+-
-> []은 빈 리스트지 0이 아님. 정수로 변환하면 0이 되는것 뿐임.
유리수: ([]++)/([]+++)
-> 2/3 - 괄호를 안해주면 +를 두 정수의 덧셈 연산자로 인식해서 문법 오류가 남
([]++).([]+++)
-> 2.3
문자열: "{[]++}{[]+++}"
-> "23" 따옴표 안에서 중괄호는 문자열로 변환하는데 쓰임.
작은따옴표도 사용 가능.
리스트: [[]+, []++, []+++]

3. 정수 연산
a + b: 덧셈
a - b: 뺄샘
a * b: 곱셈
a / b: 나눗셈
a % b: 나머지
a // b: 몫
a ** b: 거듭제곱
a+: a + 1
a-: a - 1
a*: a * 2
a/: a / 2
a%: a % 2
a//: a // 2
a**: a ** 2
++a, --a, +=, -=, *=, /=, %=, //=, **=: 님들이 아는 그대로
!a: not a
a | b: or
a & b: and
a ^ b: xor
a++, a--는 헷갈릴 수 있음 -> X
+a = a
-a = -1 * a
==, !=, >, <, >=, <=: 님들이 아는 그대로
===, !==: 타입까지 같은거 (1 !== 1.0)

10. 그런데...

11. 허용되지 않는 문자 다루기
모듈 "".!"를 임포트하자.
<""> = <!>("\"\"")
<!"> = <"">.<!">
!" 모듈의 !" 클래스를 사용하자.
이건 허용되지 않는 문자열 클래스이다.
<+> = <!">.<!">()
이번에는 !' 클래스를 사용하자.
매개변수로 유니코드값을 주자.
<-> = <!">.<!'>(<+>("{[]++++++}{[]+++++}"))
이제 변수 +에 변수 -를 더하자.
<+> += <->
그러면 +는 이제 "a"이다!
그러면
<[-]>(<+>)
와 같이 출력할 수도 있다.

12. 모듈들
"" - 문자 모듈: 기본적인 문자 다루기
"".!" - 허용되지 않는 문자 모듈: 비허용 문자 다루기
"".[-] - pretty print
+* - 기본 수학 모듈: 로그, 루트, 조합 등의 계산을 포함하는 간단한 수학 모듈
+*.+ - 기본 정수 모듈: 기본적인 정수 모듈
+*.% - 정수론 모듈: 간단한 정수론 모듈
+*./ - 유리수: 간단한 유리수 계산 등의 모듈
+*., - 실수 (.은 함수 이름 칸에 못 씀): 간단한 실수 모듈
+*.& - 진법과 불 형: 진법과 불 형에 대한 간단한 모듈
+*.! - 허수와 복소수: 복소수의 구현과 계산, 복소평면 관련 계산 등의 모듈
+*.# - 행렬: 행렬 계산 모듈
() - 함수: 기본적인 함수 모듈 (함수 여부나 일대일함수 등)
().' - 미적분: 미분, 적분의 간단한 계산 모듈
[] - 리스트 모듈 (range, zip, enumerate, 튜플, 딕셔너리, 셋 등 포함)
[].~ - 이터레이터 모듈 (이터레이터와 관련 내용)
| - 통신 모듈
|.| - 다른 언어와의 연결 모듈 (python, C/C++)
|.% - 소켓 통신 모듈
|.$ - 데이터베이스 관리 모듈 (mysql, sqlite)
|.\/ - 폴더 관리 모듈
` - 마우스 모듈
: - 키보드 모듈

14. 이게 왜 난해
이 언어는 난해라기엔 엄청난 지원과 완성률을 가진다.
기호가 약간 특이할 뿐.
근데 왜 난해?
이 언어는 개발자 공격을 하지 않는다.
이 언어는 실행기를 만드는 사람을 공격한다.
예시:
<+>+++<->
여기의 +는 ++의 +일까? 아니면 +의 +일까?
<+>+는 +1의 +일까 덧셈의 +일까?
실제로 +는 이름의 +, ++의 +, +=의 +, +의 + 등 많은 종류로 쓰인다.
구현하기 힘들겠다...

15. 구현 힌트
1. 우선순위 ++ -> +
2. 우선순위 -- -> -

4. <+>++<-> -> ++a -> a+의 순서
5. <+> + 와 같이 중간 공백이 있으면 덧셈 아니면 +1
