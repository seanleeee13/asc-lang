# 아슥 (asc) 프로그래밍 언어

- 어원: 아스키 코드 (ascii)

## 변수
`<변수명> = 값`과 같이 정의할 수 있다.

`<b> = 1 + (<a> := 3)`과 같이도 가능하다. ([참고](https://docs.python.org/ko/3/whatsnew/3.8.html#assignment-expressions))

## 값 표현
- 자연수: `[]+++`
    - `+`의 개수가 수가 된다. 따라서 위의 예시는 3이 된다.
    - `a+`와 같이 썼을 때 `+` 연산자는 1을 늘리는 연산을 하고, 빈 리스트는 정수형으로 변환했을 때 0으로 취급된다.
- 음의 정수: `[]---`
    - `-` 연산자는 1을 줄인다.
- 0: `[]+-`
    - `[]`은 빈 리스트지 0이 아니다. 정수로 변환하면 0이 되는것 뿐이다.
- 유리수: `[]+++/[]++`
    - 3/2 = 1.5가 된다.
    - 이런 계산에서 우선순위 때문에 이 언어의 실행기를 구현하는 건 매우 어려워진다.
- 실수: `[]++.[]+++`
    - 2.3이 된다.
    - (구현 난이도)+
- 문자열: `"{[]++}{[]+++}"`
    - "23"이다.
    - 따옴표 안에서 중괄호는 문자열로 변환하는데 쓰인다. ([참고](https://docs.python.org/3/reference/lexical_analysis.html#formatted-string-literals))
    - 작은따옴표도 사용 가능하다.
- 리스트: `[[]+, []++, []+++]`
    - 이것도 파이썬에서 가져왔다.

## 정수 연산
변수 값을 불러올 때는 `<a>`와 같이 `<>` 연산자를 써 줘야 하지만 여기에서는 생략

- `a + b` - 덧셈
- `a - b` - 뺄샘
- `a * b` - 곱셈
- `a / b` - 나눗셈
- `a % b` - 나머지
- `a // b` - 몫
- `a ** b` - 거듭제곱
- `a+` - `a + 1`
- `a-` - `a - 1`
- `a*` - `a * 2`
- `a/` - `a / 2`
- `a%` - `a % 2`
- `a//` - `a // 2`
- `a**` - `a ** 2`
- `++a`, `--a` - 증감 연산자 ([참고](https://namu.wiki/w/C(%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D%20%EC%96%B8%EC%96%B4)/%EB%AC%B8%EB%B2%95#s-5.3))
    - `a++`, `a--`는 `a+`, `a-`와 혼동의 여지가 있다.
    - `(++a)-`, `(--a)+`로 사용하자.
- `+=`, `-=`, `*=`, `/=`, `%=`, `//=`, `**=` - `a = a + b`와 같이 계산 후 대입한다.
- `+:=`, `-:=`, `*:=`, `/:=`, `%:=`, `//:=`, `**:=` - +=과 :=을 합친 것이다. `(a := a + 1)`
- `!a`: a가 0이면 `!a`는 1, a가 0이 아니면 `!a`는 0이다.
- `a | b`, `a & b`, `a ^ b`, `~a`: 비트 연산자 ([참고](https://namu.wiki/w/C(%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D%20%EC%96%B8%EC%96%B4)/%EB%AC%B8%EB%B2%95#s-5.4))
- `+a` - `1 + a`
- `-a` - `-1 * a`
- `==`, `!=`, `>`, `<`, `>=`, `<=`: 일반적인 비교 연산자
- `===`, `!==`: 타입까지 같아야 하는 비교 연산자 (1 !== "1") ([참고](https://namu.wiki/w/%3D%3D%3D))

## 문자열, 리스트 연산
- `a + b`, `a * n` - 문자열/리스트 이어붙이기, 문자열/리스트 반복
    - `+=`, `+:=`, `*=`, `*:=`도 가능하다.
- `*a`: 리스트 언패킹 ([참고](https://docs.python.org/ko/3.14/tutorial/controlflow.html#tut-unpacking-arguments))

## 불 연산
- `a | b`, `a & b`, `a ^ b`, `~a`: 각각 or, and, xor, not

`<함수명>(매개변수)`로 실행할 수 있다.

### 내장 함수 목록
- `<[-]>`: 출력
- `<[+]>`: 입력 (한 줄)

string split 등을 모두 지원 안 하는 것 같지만...

### 사용자 지정 함수
- `<함수명> = (매개변수) => {함수 내용}`으로 사용자 지정 함수를 만들 수 있다. ([참고](https://namu.wiki/w/JavaScript/%EB%AC%B8%EB%B2%95#s-6.1))

    - 데코레이터: `<함수명> = #<데코레이터> (매개변수) => {함수 내용}`
        - 여러 개를 중첩할 수 있으며, 중첩 시 뒤쪽에 있는 것부터 실행된다.
    - 타입 어노테이션: 주석으로 처리하기

## 제어문
### if 문
- `<?>(조건문, 조건이 맞으면 실행할 함수, 조건이 틀리면 실행할 함수)`으로 사용할 수 있다.
- 예시: `<?>(a < 1, () => {a *= -2}, () => {a *= 2})`
- 조건이 틀리면 실행할 함수는 없어도 된다.
### for 문
- python식 for: `<*?>(반복 범위, 반복할 함수)`
- C++식 for: `<*?>(초기식, 조건식, 증감식, 반복할 함수)`
### while 문
- `<**>(조건 함수, 반복할 함수)`
- 예시: `<**>(() => {<^>(a < 1)}, () => {<[-]>(a); --a})`
    - 파이썬과 같이 세미콜론은 선택적 허용임

## 허용되지 않는 문자 다루기
모듈 `"".!"`를 임포트하자.
```
<""> = <!>("\"\"")
<!"> = <"">.<!">
```
`!"` 모듈의 `!"` 클래스를 사용하자.
이건 허용되지 않는 문자열 클래스이다.
```
<!+> = <!">.<!">()
```
이번에는 `!'` 클래스를 사용하자.
매개변수로 유니코드값을 주자.
```
<!-> = <!">.<!'>(<+>("{[]++++++}{[]+++++}"))
```
이제 변수 `!+`에 변수 `!-`를 더하자.
```
<!+> += <!->
```
그러면 `!+`는 이제 "a"이다!

그러면
```
<[-]>(<!+>)
```
와 같이 출력할 수도 있다.

## 모듈들
- `""` - 문자 모듈: 기본적인 문자 다루기
- `"".!"` - 허용되지 않는 문자 모듈: 비허용 문자 다루기
- `"".[-]` - pretty print
- `+*` - 기본 수학 모듈: 로그, 루트, 조합 등의 계산을 포함하는 간단한 수학 모듈
- `+*.+` - 기본 정수 모듈: 기본적인 정수 모듈
- `+*.%` - 정수론 모듈: 간단한 정수론 모듈
- `+*./` - 유리수: 간단한 유리수 계산 등의 모듈
- `+*.,` - 실수 (.은 모듈 이름 칸에 못 씀): 간단한 실수 모듈
- `+*.&` - 진법과 불 형: 진법과 불 형에 대한 간단한 모듈
- `+*.!` - 허수와 복소수: 복소수의 구현과 계산, 복소평면 관련 계산 등의 모듈
- `+*.#` - 행렬: 행렬 계산 모듈
- `()` - 함수: 기본적인 함수 모듈 (함수 여부나 일대일함수 등)
- `().'` - 미적분: 미분, 적분의 간단한 계산 모듈
- `().{}` - 클래스: 기본적인 클래스와 init, operator 등
- `().{}.*{}` - abstract 클래스
- `[]` - 리스트 모듈 (range, zip, enumerate, 튜플, 딕셔너리, 셋 등 포함)
- `[].~` - 이터레이터 모듈 (이터레이터와 관련 내용)
- `|` - 통신 모듈
- `|.|` - 다른 언어와의 연결 모듈 (python, C/C++)
- `|.%` - 소켓 통신 모듈
- `|.$` - 데이터베이스 관리 모듈 (mysql, sqlite)
- `|.\/` - 폴더 관리 모듈
- `` ` `` - 마우스 모듈
- `:` - 키보드 모듈

## 이게 왜 난해
이 언어는 난해라기엔 엄청난 지원과 완성률을 가진다.

기호가 약간 특이할 뿐.

근데 왜 난해할까?

이 언어는 개발자 공격을 하지 않는다.

이 언어는 실행기를 만드는 사람을 공격한다.

예시:

`<+>+++<->`

여기의 `+`는 `++`의 `+`일까? 아니면 `+`의 `+`일까?

`<+>+`는 `+1`의 `+`일까 덧셈의 `+`일까?

실제로 `+`는 이름의 `+`, `++`의 `+`, `+=, `+:=`의 `+`, `+`의 `+` 등 많은 종류로 쓰인다.

구현하기 힘들겠다...

## 구현 힌트
1. 우선순위 ++ -> 증가 +
2. 우선순위 -- -> -
3. 나머지는 다른 언어와 유사
4. <+>++<-> -> ++a -> a+의 순서
5. <+> + 와 같이 중간 공백이 있으면 덧셈 아니면 +1
6. +(++a)+ + +b는 (+((a := (a + 1)) + 1)) + (+b)이다.

겁나 복잡하네
